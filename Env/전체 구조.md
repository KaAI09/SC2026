[개발 환경 전체 구조]

대회장에서는 팀원 5명이 각자 개인 노트북을 가져와 Local PC로 사용하고, D3-G 보드 및 연구실 Server PC와 연결하여 개발 진행

[1. Local PC - 팀원 개인 노트북 5대]

* 코드 작성 및 수정
* 팀원별 개인 브랜치에서 개발 진행
* 수정사항을 공용 Git Repository에 push

[2. D3-G Board - 스케일카]

* D3-G 내부에서는 코드 직접 수정하지 않음
* Git에서  최신 코드 pull
* 코드 빌드 후 프로그램(노드) 실행
* 실차 주행 및 최종 동작 테스트 진행
* pull, build, run 용도로만 활용

[3. Control PC 역할]

* Windows 노트북 사용자는 하드웨어 드라이버와 MobaXterm 사전 설치 필요

* 별도의 Control PC를 준비하지 않고 팀원의 Windows 노트북 중 1대를 활용
* MobaXterm을 통한 Serial 연결을 통해 D3-G Wi-Fi 및 네트워크 설정
* 테스트 과정 실시간 로그 및 주행 영상 확인
*  USB-C 연결을 통한 대용량 파일 전달

[4. Server PC - 연구실 컴퓨터]

* Linux 기반 GPU 컴퓨터
* 빠른 모델 학습이 필요한 경우에만 활용
* 대회장에서 원격 접속 가능하도록 설정 완료

[Git Repository 운영 방식]

* 모든 코드는 하나의 공용 Repository에서 관리
* 팀원별 개인 브랜치에서 코드 작성 및 수정
* 최종 테스트 완료한 버전만 main 브랜치에 반영
* D3-G에서는 직접 코드 수정 및 push 진행하지 않음

[기본 개발 흐름]

Local PC에서 코드 수정
→ 개인 브랜치에 commit 및 push
→ D3-G에서 테스트 브랜치 pull
→ D3-G에서 build 및 run
→ Control PC 역할의 Windows 노트북에서 로그와 주행 영상 확인
→ 주행 성공(개선) 시, 최종 버전을 main 브랜치에 반영

[GPU 통한 모델 학습 소요 발생 시]

개인 노트북에서 Server PC에 SSH 접속
→ GPU를 이용해 모델 학습
→ 학습된 모델을 D3-G로 전달
→ D3-G에서 모델 추론 및 실차 테스트

[비고]

* MobaXterm은 D3-G Wi-Fi 설정에만 활용 (SSH 연결 전)
* USB-C 연결 대용량 파일 직접 전달 시 활용 (예: 학습된 모델 파일)
